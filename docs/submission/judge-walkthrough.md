# Judge Walkthrough — Provision

## 60-Second Impression
A judge opening Provision sees a clean mobile app for SNAP recipients. Onboarding takes 30 seconds. The Home screen immediately shows deadlines and risk level. No clutter, no jargon.

## 3-Minute Technical Deep Dive

### What the Judge Should Look For

1. **AI Architecture** (`docs/submission/ai-architecture.md`)
   - Hybrid: deterministic rules + Gemini explanation
   - Rules engine owns all hard numbers (USDA FY2026 thresholds)
   - Gemini only writes plain-language explanations from retrieved snippets
   - Merge layer: deterministic values override AI values

2. **Responsible AI** (`docs/submission/responsible-ai.md`, `docs/submission/human-in-the-loop.md`)
   - Never says "you qualify" — always "you may qualify" or "likely eligible"
   - Every AI result shows: disclaimer, citations, confidence, caseworker phone
   - Prompt injection resistance: user text treated as untrusted data
   - Human-in-loop: county caseworker makes all final decisions

3. **Testing** (`backend/tests/`)
   - 127 automated tests
   - Domain accuracy: every SNAP value verified against USDA FY2026
   - Security: CORS, secrets, prompt injection, oversized payloads
   - Fallback: all routes work with Gemini unavailable

4. **Demo Reliability** (`docs/submission/demo-script.md`)
   - 3 demo personas available via `/demo/scenarios`
   - App works offline for onboarding + home
   - AI screens degrade gracefully when Gemini is down

## What Makes This Different

- Not a benefits directory — it interprets situations ("I got a raise") into actions ("report within 10 days")
- Not a chatbot — deterministic rules ensure accuracy, AI ensures understanding
- Not generic — California (SAR-7) and Texas (QR-7) have different flows
- Not unsafe — every output cites policy sources and routes to a human caseworker

## Questions a Judge Might Ask

**"Why AI instead of just a rules engine?"**
A rules engine can compute deadlines but can't interpret "my roommate moved out" or read a photographed notice. AI handles natural language understanding while the rules engine ensures accuracy.

**"How do you prevent hallucinated deadlines?"**
Deadlines are hardcoded constants in `rules_engine.py`, not generated by Gemini. The merge layer prefers deterministic values over AI values. Tests verify this (`test_deterministic_wins_over_gemini`).

**"What if Gemini is down?"**
Every route returns a safe deterministic fallback. The `ai_explanation_unavailable` flag tells the frontend to show "AI explanation is temporarily unavailable" while still showing the deadline and caseworker phone.

**"Is the eligibility check accurate?"**
Every income threshold is verified against USDA FNS FY2026 published values in `test_domain_accuracy.py`. 26 tests compare all 8 household sizes for gross limits, net limits, and max allotments.
