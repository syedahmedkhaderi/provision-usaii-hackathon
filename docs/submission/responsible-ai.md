# Responsible AI Statement — Provision

## One Realistic Risk

**Risk: Over-reliance on AI eligibility interpretation.**

A SNAP recipient using Provision may over-rely on an AI-generated explanation and miss a critical deadline, misunderstand their eligibility, or fail to contact their caseworker — causing the exact procedural benefit loss the app is trying to prevent. If the AI says "you likely need to report this within 10 days" but the user treats this as certain and doesn't verify, and the actual rule requires 5 days for their specific situation, the user loses benefits.

## One Concrete Mitigation

**All hard numbers are deterministic. Gemini only writes plain-language explanations.**

- Income thresholds (130% FPL gross limits, 100% FPL net limits) are hardcoded from USDA FNS FY2026 published values, not generated by AI
- Maximum benefit allotments are hardcoded from USDA FNS FY2026 COLA tables
- Reporting deadlines (10 days for changes, 90 days for fair hearings) are hardcoded constants
- SAR-7 timing (6 months), recertification timing (CA: 12 months, TX: 6 months) are deterministic calendar calculations
- Every AI output includes:
  - Citations with actual policy section numbers (e.g., "CalFresh Policy Manual §63-503.44")
  - Disclaimer: "Guidance only. Not legal advice. Verify with your caseworker."
  - Confidence level (high/medium/low)
  - Caseworker phone number (CA: 1-877-847-3663, TX: 2-1-1)
- Low-confidence outputs route to caseworker with a pre-generated call script
- The app uses "you may qualify" and "likely eligible" — never "you qualify" or "eligible"
- User-provided text (change descriptions, notice content) is treated as untrusted data in LLM prompts, with explicit instructions to the model not to follow embedded commands

## Human-in-the-Loop Boundary

**The county caseworker or state benefits agency makes all final decisions.** Provision does not:

- Determine final eligibility (gross/net income tests, resource tests, categorical eligibility)
- Set benefit amounts
- Approve or deny applications
- Adjudicate disputes or fair hearing requests
- Authorize benefit changes
- Replace official agency communication

Provision helps the user:
- Understand what a notice means
- Know whether a change likely needs reporting
- Prepare for a caseworker call with a script
- Track upcoming deadlines
- Access recovery options if benefits are disrupted

**Why a human must remain in control:** An AI cannot verify income documentation (pay stubs, award letters), confirm household composition, adjudicate disputes between the agency and recipient, or account for state-specific edge cases (expedited screening, categorical eligibility, ABAWD waivers). Pretending it could would create false confidence and potential harm.

## Technical Safeguards

1. **Deterministic rules override AI output** — if Gemini returns `likely_eligible: false` but the rules engine says income is under threshold, the rules engine wins
2. **Prompt injection resistance** — system prompts explicitly treat user input as untrusted data, not instructions
3. **Safety-blocked responses handled gracefully** — if Gemini blocks content for safety reasons, the app returns a fallback directing the user to call their caseworker instead of crashing or disabling AI globally
4. **No API keys in frontend** — Gemini keys are backend-only, never bundled into the mobile app
5. **Input validation** — all API payloads are validated (max length, range constraints, typed schemas) to prevent abuse
6. **Graceful degradation** — if Gemini is unavailable, every route returns a safe deterministic fallback with the caseworker phone number
